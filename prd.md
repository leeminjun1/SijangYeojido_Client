개요
한 줄 정의
시장여지도: 전통시장 실시간 특가·예약 플랫폼
제품 목표
전통시장의 파편화된 정보와 낮은 거래 전환 문제를 해결하기 위해 2D 시장 지도, 점포 카드(신뢰 정보), 실시간 특가(빨간 핀), 15분 예약금 결제·픽업을 하나의 흐름으로 묶어 ‘오프라인 픽업 거래’를 늘린다.
North Star(성공의 기준): 주간 픽업 완료 거래 수
핵심 퍼널(분모/분자 정의 고정)
* 탐색: 시장 선택→지도 첫 렌더 완료→검색/필터 실행→핀 노출→핀 클릭→점포 카드 열림
* 전환: 점포 카드→딜 상세→예약 시작→결제 성공→픽업 완료
운영 원칙(요약)
* 정보는 ‘최신성(마지막 수정일/출처/승인)’이 보이는 형태로 제공한다.
* 허위/품절/부적절 콘텐츠는 신고→임시 조치→검수→제재로 품질을 유지한다.
* 거래는 TTL(15분)과 예약금으로 노쇼를 줄이고, 상태 원장/감사 로그로 분쟁·정산을 통제한다.
파일럿 원칙(단일 시장)
* 수치가 확정되지 않아도 운영 가능하도록: 승인 큐 운영 시간, 신고 1차 확인 SLA, 데이터 갱신 책임(상인/운영)을 먼저 합의하고 시작한다.

배경
전통시장은 가격/신선도의 강점에도 불구하고 디지털 접점과 실시간 정보가 부족해 탐색·방문·구매 전환이 낮다. 특히 시장 내부는 구역/점포명이 비표준이고, ‘실제 영업 여부/재료 소진/결제수단/당일 입고/마감 특가’ 같은 핵심 정보가 현장에 묻혀 있다.
지도 앱은 시장 내부(구역·점포·인프라)를 촘촘히 다루기 어렵고, 커뮤니티는 정보가 파편화되며, 배달앱은 오프라인 방문 전환이 중심이 아니다. 따라서 “시장 내부 탐색 + 신뢰 정보 + 실시간 특가 + 예약금 결제 + 픽업”을 한 흐름으로 묶어야 거래가 완결된다.
운영 관점에서 ‘정보 최신성’과 ‘허위/품절 분쟁’을 통제할 정책이 없으면 품질이 급락한다. 이 제품은 표시 규칙(최신성 배지), 신고 처리 SLA, 제재/해제 조건, 환불/귀속 로직, 감사 로그를 제품 문서에 포함해 파일럿부터 확장 가능한 운영 체계를 만든다.
초기 파일럿은 단일 시장을 대상으로 운영·정책을 안정화하고(데이터 승인, 신고 SLA, 제재/환불 로직), 복수 시장 확장을 전제로 데이터 구조(도면 버전, 구역 좌표, 점포 승인)를 갖춘다.

핵심 가치
사용자 문제
* 복잡한 시장 구조와 비표준 점포명으로 탐색이 어렵다
* 결제수단/실제 영업/재료 소진 등 핵심 정보 비대칭
* 당일 입고/마감 특가 정보의 실시간성 부재
* 마감 재고·떨이 정보의 비가시성으로 기회손실 발생
* 예약 노쇼 리스크로 상인이 예약 기능을 기피
* 시장 정보가 외부 커뮤니티에 파편화
문제의 핵심은 ‘정보의 최신성(언제/누가/어떻게 갱신했는지)’과 ‘실시간 거래 전환(딜→예약→픽업)’이 같은 시스템 안에서 측정/통제되지 않는다는 점이다.
추가로, 시장 내부 네트워크 불안정(저전파/오프라인) 환경이 많아 ‘기본 탐색 폴백’이 없으면 사용 경험이 급격히 나빠진다.
해결 방식
1. 탐색(2D 지도) — 같은 결과 집합 유지
* 2D 마이크로 맵으로 구역·점포·품목 기반 탐색
* 인프라 POI(화장실/주차장/ATM/출입구) 상시 노출
* 검색/필터/핀/리스트/바텀시트(점포 카드)의 상태를 단일 기준으로 동기화해, 항상 ‘같은 결과 집합’을 보게 한다.
* 저전파/오프라인 대비: 캐시/폴백으로 최소 탐색(POI/리스트)을 유지하고, 갱신 시각과 폴백 상태를 노출한다.
1. 신뢰 정보(점포 카드) — 최신성/출처/정책
* 결제 태그, 영업 상태/휴무, 최근 수정일, 대표 사진/상품을 구조화
* 정보 출처 우선순위(운영 확인>상인 입력>사용자 제안)로 필드 충돌을 해소
* 최신성 배지(갱신 필요) 노출 규칙과 배지 클릭 후 행동(수정 제안/신고)을 제공
* 자유 메모는 금지어/개인정보 필터를 적용하고, 신고 시 임시 숨김 조건을 운영 정책으로 제공
1. 실시간 특가(빨간 핀) + 반경 푸시 — 과도 알림/허위 딜 통제
* 상인이 ‘떨이 시작’하면 딜이 live로 전환되어 지도/홈에 빨간 핀으로 노출
* 반경 기반 푸시(기본 1km), 중복 발송 제한(쿨다운/빈도 제한), 야간 제한으로 과도 알림을 방지
* 허위/반복 딜 제재(경고→노출 제한→정지)와 해제 조건을 운영 정책으로 제공
1. 예약금 결제 + 15분 픽업 — 상태 원장/감사 로그
* 딜 상세에서 예약금/전액 결제 선택
* 결제 성공 시 15분 TTL 시작, 픽업 코드(4자리/QR)로 수령 확인
* TTL 만료/점포 귀책 취소/분쟁 시 환불·귀속 로직과 감사 로그로 정산 가능
* 만료/환불/픽업 완료는 중복 처리 방지(idempotent) 기준을 명시
1. 상인/운영 도구 — 승인 큐/SLA/감사
* 상인센터: 30초 내 딜 발행/종료, 예약 큐/픽업 처리, 전일 복제
* 운영콘솔: 데이터 승인, 신고/분쟁 처리, 강제 종료/환불 승인, 감사 로그/내보내기
* 핵심 이벤트 트래킹/대시보드로 KPI를 일관된 분모/분자로 관리

차별점
* 지도 앱 대비: 시장 ‘내부’ 탐색(구역/POI/점포/품목)과 점포 카드 신뢰 정보에 최적화
* 커뮤니티 대비: 정보 구조화(필드/정책) + 신고/승인 워크플로우로 최신성을 통제
* 배달앱 대비: 오프라인 방문·픽업 전환 중심(근거리 실시간 딜 + 15분 픽업)
추가 차별점(운영 가능성)
* ‘빨간 핀 + 예약금 결제’로 거래를 완결시키고, 운영 정책(임시 숨김·제재·환불·감사 로그·SLA)로 품질을 유지한다.
* 저전파/오프라인을 고려한 캐시/폴백을 제품의 기본 요구사항으로 포함해 시장 환경 적합성을 높인다.

타겟 및 시나리오
타겟 사용자
* 생활권 실속형 소비자(시장 반경 1~3km, 퇴근길 장보기): 빠른 탐색, 실패 없는 첫 방문, ‘오늘만’ 특가 발견
* 시장 초행 방문자(구역·인프라 안내 필요): 구역/POI 기반 길찾기, 점포 신뢰 정보
* 상인(간편 홍보/떨이/픽업 처리 필요): 30초 내 딜 발행/종료, 예약 큐 처리, 노쇼 방지(예약금)
* 시장 운영 주체/상인회(정보 품질·성과 관리): 신고/분쟁 처리, 데이터 승인, 정산/감사 로그 기반 관리
역할별 핵심 행동(요약)
* 사용자: 탐색(지도/검색)→점포 카드 확인→딜/예약→픽업
* 상인: 딜 생성/종료→예약 확인→픽업 처리
* 운영자(시장 매니저): 신고/정보 승인→강제 종료/환불 승인
* 관리자(슈퍼어드민): 정책/권한/템플릿 관리
권한 원칙(간단)
* 사용자는 탐색/신고/수정 제안 가능(승인 권한 없음)
* 상인은 본인 점포의 딜 생성/종료/픽업 처리 가능
* 운영자는 신고 처리, 딜 강제 종료/환불 승인, 데이터 승인 가능
* 관리자는 정책(푸시 템플릿/임계치/권한) 관리 가능

사용 시나리오
사용자(탐색: 지도→점포 카드)
* 시장 선택 → 지도 첫 화면 렌더(저전파 시 캐시 폴백) → 품목 검색/필터 → 핀/리스트 탐색 → 핀 탭 → 점포 카드(바텀시트)에서 결제/영업/사진/대표 품목/최신성 배지 확인 → 길찾기/즐겨찾기/신고/딜 보기
* 예외: 0건 결과 폴백(동의어→인접 품목→전체 보기), 점포 정보 로딩 실패(재시도/문의), 휴무/영업 불일치(신고/임시 조치)
사용자(딜/픽업)
* 반경 1km 푸시 수신(권한 미동의 시 즐겨찾기 시장 기준 대체) → 빨간 핀 상세 → 예약 시작 → 결제(성공/실패/결제 확인 중) → 15분 TTL 진행(만료 5분 전 알림) → 픽업 코드 제시 → 픽업 완료
* 예외: 매진/만료/휴무는 즉시 상태 치환, TTL 만료 시 귀속/통지, 점포 귀책 취소 시 자동 환불
상인
* 떨이 시작(필수 입력: 상품/가격/수량/종료시각) → 딜 노출/푸시 → 예약 확인 → 픽업 처리(코드/QR) → 정산 로그 확인
* 예외: 네트워크 불안정 시 임시 저장/재전송, 반복 허위 딜 시 생성 제한
운영
* 허위/품절/부적절 콘텐츠 신고 접수 → 임시 숨김/표시 유지(규칙) → 증빙 확인 → 딜 강제 종료/환불 승인/제재 → 감사 로그 기록
파일럿 운영 가정(초기 합의용)
* 단일 시장에서 점포/품목/딜 빈도를 측정 가능한 목표치로 설정하고, ‘정보 갱신일 경과 시 배지’ 정책과 신고 SLA를 준수한다.
* 승인 큐 운영 시간/담당자 체계를 먼저 합의해, 사용자 제안과 상인 수정이 누적되지 않게 한다.

성공 지표
핵심 KPI
North Star: 주간 픽업 완료 거래 수
퍼널(분모/분자 정의를 팀 공통으로 관리)
* 지도 탐색
    * 지도 첫 렌더 성공률(분모: 지도 진입, 분자: N초 내 렌더 완료)
    * 지도 폴백 발생률(분모: 지도 진입, 분자: 캐시/리스트/POI 폴백 진입)
    * 폴백 복구율(분모: 폴백 진입, 분자: 이후 정상 지도 렌더 복귀)
    * 품목 검색 이용률(분모: 지도 진입, 분자: 검색 실행)
    * 필터 적용률(분모: 지도 진입, 분자: 필터 적용)
    * 핀 클릭률(분모: 핀 노출 사용자 수, 분자: 핀 클릭 사용자 수)
    * 점포 카드 열림률(분모: 핀 클릭, 분자: 카드 열림)
    * 점포 카드 로딩 실패율(분모: 카드 열림, 분자: 로딩 실패)
* 전환
    * 딜 상세 진입률(분모: 점포 카드 열림, 분자: 딜 상세 진입)
    * 예약 시작률(분모: 딜 상세, 분자: 예약 시작)
    * 결제 성공률(분모: 예약 시작, 분자: 결제 성공)
    * 픽업 완료율(분모: 결제 성공, 분자: 픽업 완료)
    * TTL 만료율(분모: 결제 성공, 분자: TTL 만료)
* 품질/운영
    * 신고율(분모: 점포 카드/딜 상세 노출, 분자: 신고)
    * 임시 숨김 비율(분모: 신고, 분자: 임시 숨김 처리)
    * 운영 SLA 준수율(분모: 신고, 분자: SLA 내 1차 확인)
    * 환불률/분쟁 건수
    * 이벤트 누락률(분모: 주요 화면 진입, 분자: 이벤트 미수신)
* 리텐션
    * 푸시 오픈율, 재방문율, 즐겨찾기 재조회율

리스크/이슈
* 정보 최신성 저하: 갱신일 표시·배지·승인 워크플로우·책임 분담(상인/운영)으로 통제
* 상인 참여 저조: 버튼형 상인 UX, 전일 복제, 현장 교육, 딜 성과 리포트 제공
* 허위 특가/품절 분쟁: 종료시각/수량 필수, 신고 SLA, 단계별 제재(경고→노출 제한→정지), 증빙 요구
* 정산/환불 복잡성: 상태 원장/감사 로그, 중복 처리 방지(idempotent), PG webhook 동기화
* 운영 비용 증가: 자동 만료/알림, 일괄 처리, 경고 규칙
* 법무/PG 검토 지연: 예약금 구조 단순화, 환불/귀속 정책 사전 합의 및 문서화
